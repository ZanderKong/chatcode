# 用 Bento 方式写博客：把信息切成小方块

过去写博客，总是被「长文」的形式绑住：一次写不完就搁置，或者为了填满篇幅而堆砌段落。Bento 风格给了我新的自由——每个模块都是一块轻量的方块。

## 选择卡片的理由

1. **聚焦单一主题**：一张卡片只回答一个问题，写起来更快。
2. **方便重排**：内容之间的顺序可以随时调整，不影响阅读。
3. **渐进式更新**：今天补全一段，明天换一个标题，持续改进不需要「大修」。

## 如何组织模块

- 用文件夹存文章：`content/posts` 放 Markdown，`posts.json` 记录元数据。
- 首页模块引入 `content/home` 下的描述文件，让读者理解当前视图。
- 在 `script.js` 里定义卡片布局，按模块加载对应的 markdown。

```ts
// 简化的加载逻辑
const response = await fetch('content/posts/posts.json');
const posts = await response.json();
```

## 结语

把大问题拆成小方块，再把方块拼出自己的宇宙。你也可以试试。
